# Angular — Standards Reference

## Status

Angular stack standards are not yet fully defined for this project.

Apply general Angular best practices:
- OnPush change detection on all components
- Reactive forms over template-driven forms
- Smart/dumb component separation
- Services for state and data access
- RxJS for reactive patterns; avoid nested subscriptions
- Standalone components (Angular 14+)
- Signal-based reactivity (Angular 17+) where available

Standards will be added directly to the relevant skills (e.g. `dev-frontend-ui-engineering`, `dev-api-and-interface-design`) when the stack is formally adopted.
